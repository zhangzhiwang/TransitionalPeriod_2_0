| 对象 | 规范 | 示例 |
|------|------|------|
| 库名 | 全小写 + 下划线 | `cursor_website` |

| 规划功能 | 表 | 说明 |

| 网站首页 | `site_config`、`home_feature`、`news` | 文案配置 + 亮点 + 最新新闻 |

## 3. ER 关系概览

```text
site_config（独立键值）
home_feature（独立）
about_section / about_timeline（独立）
product（独立）
news（独立）
contact_message（独立）
job 1──* job_application
friend_link（独立）
page_view_daily（独立，按 view_date + page_path 唯一）
guestbook（独立）
```

说明：投递记录通过 `job_id` 逻辑关联职位；为保持与现网脚本一致、便于灌数，未强制外键约束。

## 4. 表字段要点

### 4.1 site_config

键示例：`home.brandName`、`home.heroHeadline`、`contact.phone`、`contact.email` 等。

### 4.2 product

- `featured`：旗舰/重点全宽展示。
- `features_json` / `specs_json`：详情「特性卷轴」「技术规格」，避免过早拆子表。

### 4.3 news

- `featured`：头条；`view_count`：详情阅读累加，供统计页聚合。

### 4.4 contact_message vs guestbook

- 联系我们：商务/客服咨询（姓名、邮箱、主题、内容）。
- 网站留言：社区留言板（昵称、内容、审核展示、可回复）。

### 4.5 friend_link

- `status=published` 前台展示；申请写入 `pending`。
- `group_name`：合作伙伴 / 行业资源等分组。

### 4.6 page_view_daily

- `(view_date, page_path)` 唯一；过滤器或 `POST /api/stats/pv` 按天累加。
- 今日/本月访问：对日期范围 `SUM(pv_count)`；热门页面：按 `page_path` 聚合。

### 4.7 job_application

- `resume_path`：仅存文件路径，文件落本地或 OSS。

## 5. 脚本与建库

| 文件 | 用途 |

```sql
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;
```
