# CursorOnlineShoppingMall 前后端接口文档

> 依据：`需求文档`、`项目架构设计文档`  
> 技术栈：Spring Boot + MyBatis-Plus + JSP + MySQL  
> Base URL：`http://localhost:8081/cursorMall`  
> （由 Curosr 自动生成）

---

## 1. 约定说明

### 1.1 接口分类

| 类型 | 路径前缀 | 说明 |
|------|----------|------|
| 页面接口 | 无统一前缀 | 返回 JSP 视图（或重定向），供浏览器直接访问 |
| 数据接口 | `/api/**` | 返回 JSON，供页面 AJAX 调用 |

### 1.2 统一响应结构（JSON）

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | `0` 成功；非 `0` 失败 |
| message | string | 提示信息（失败时给前端展示） |
| data | object / array / null | 业务数据 |

### 1.3 常用错误码

| code | 含义 |
|------|------|
| 0 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 冲突（如用户名已存在） |
| 500 | 服务器错误 |

### 1.4 登录态

- 登录成功后由服务端建立 **Session**（Cookie：`JSESSIONID`）。
- 需登录的接口：未登录返回 `code=401`，页面接口则 **302 重定向** 到 `/login`。
- 密码落库须加密（MD5 / SHA-256 等），接口传输可用 HTTPS；本文档示例为明文入参，落库由后端加密。

### 1.5 需登录范围（对应需求 0.3）

首页、分类、购物车、我的、商品详情、分类列表、分类商品列表、订单、收货地址、订单确认等页面及其关联 `/api/**` 接口，均需登录。

## 3. 认证与用户

### 3.1 发送注册验证码

- **POST** `/api/auth/send-register-code`
- **登录要求**：否
- **说明**：向邮箱发送 **6 位数字**验证码，有效期 5 分钟；同一邮箱 60 秒内不可重复发送。

**请求体**

```json
{
  "email": "zhangsan@example.com"
}
```

**成功响应**

```json
{ "code": 0, "message": "验证码已发送，请查收邮箱", "data": null }
```

---

### 3.2 用户注册

- **POST** `/api/auth/register`
- **登录要求**：否
- **说明**：校验邮箱验证码通过后写入 `mall_user`；成功后跳转登录页。

**请求体**

```json
{
  "username": "zhangsan",
  "password": "abc123456",
  "confirmPassword": "abc123456",
  "email": "zhangsan@example.com",
  "verifyCode": "123456"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| username | 是 | 用户名，唯一 |
| password | 是 | 密码 |
| confirmPassword | 是 | 确认密码，须与 password 一致 |
| email | 是 | 接收验证码的邮箱 |
| verifyCode | 是 | 邮件中的 6 位数字验证码 |

**成功响应**

```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "userId": 1001,
    "username": "zhangsan"
  }
}
```

**失败示例**

```json
{ "code": 409, "message": "用户名已存在", "data": null }
```

```json
{ "code": 400, "message": "验证码错误", "data": null }
```

```json
{ "code": 400, "message": "两次密码不一致", "data": null }
```

### 3.6 微信登录授权（可选扩展）

> 需求 4.1：头像与昵称通过微信登录授权获得。账号密码登录可与微信绑定并存。

- **GET** `/api/auth/wechat/authorize`  
  重定向到微信授权页。
- **GET** `/api/auth/wechat/callback?code={code}&state={state}`  
  授权回调：换取 openid/用户信息，写入 Session，绑定或更新 `nickname`、`avatarUrl`，再重定向「我的」或首页。

## 11. 前端调用示例（原生 JS）

```javascript
// 加入购物车示例
fetch('/cursorMall/api/cart/items', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'same-origin',
  body: JSON.stringify({ productId: 101, quantity: 1 })
})
  .then(function (res) { return res.json(); })
  .then(function (body) {
    if (body.code === 0) {
      alert(body.message);
    } else if (body.code === 401) {
      location.href = '/cursorMall/login';
    } else {
      alert(body.message || '操作失败');
    }
  });
```
